Hospital Messenger System

Timeline: September 2016 – December 2016

Tech: Object-Oriented Programming, Java

This project involved designing and implementing a secure messaging system for hospital staff. I began by gathering user requirements from the TA and professor (acting as clients) and writing acceptance tests and unit tests to define expected system behavior.

Users such as doctors, nurses, and administrators register with an ID and name. The system supports group-based messaging with strict privacy rules: users can only send messages to groups they belong to, and can only read messages from their registered groups. Messages have states (new/old), and users can delete their old messages.

The system includes commands for managing users, groups, registrations, and messages (e.g., add_user, add_group, register_user, send_message, read_message, delete_message). It also supports queries such as list_users and list_groups, sorted as required. New and old messages are maintained in sorted lists for each user.

I implemented the system in Java using Maps, Sets, Lists, and Iterators to model the abstract state and enforce data integrity. This project strengthened my understanding of requirements elicitation, use cases, acceptance testing, and building secure, scalable software systems.
