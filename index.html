PROJECTS

September 2016 - April 2017

AMD application
Applied Technical skills: OOP, JAVA, Android, Firebase

This was a great in-class project which helped me dive further into mobile app development. My project was very critical since it included dealing with a serious medical condition. AMD means age-related macular degeneration. It is deterioration of the macula, which is the small central area of the retina of the eye that controls visual acuity. It produces a slow, painless loss of vision and the patient loses central vision. They can not see fine details, whether they're looking at something close or far.

I created the app using Android studio. I also used a Firebase database to store the patient information, test results and test patterns. The patients first had to register and then login in order to use the application. The mobile app included showing random straight lines of different sizes with a red dot in the center. The AMD patients may see some of the lines as wavy or blurred, with some dark areas at center. The app included displaying lines one after another and the patient will tap/voice "yes" if they see the line straight or "no" if they don't see a straight line. The test results were stored in the Firebase database for later use and also displayed in the app.

This was a very memorable experience for me. I had many failures along the way but I was able to achieve the goal and meet the deadline for the project.

September 2016 - December 2016

Hospital Messenger System
Applied Technical Skills: OOP, JAVA

This project included implementing a secure messenger system that can be used in a hospital. The first step to creating this system was to write acceptance tests and actual tests. I also met with the clients(in this case the TA and the professor) to derive user requirements.

The users(e.g. doctors, nurses, administrators) first have to register to the system using an ID and a name. The messenger system must support the ability of users to send text messages. There are also groups and users are members of groups and when they send a message it is always to a group, with the privacy condition that only members of that group may read the message. Users may become members of any number of groups. Users may send messages only to the members of a group they are registered in. Users may only access/read a message from a group they are registered in. Once a message is read by the recipient, its status changes from new to old. Users may delete their old messages.

The system provides the following queries: list_users (ID → name) and list_groups (ID → name), both sorted by name. Components of the abstract state are always sorted by ID, except for the list of users, or lists of groups (as above). Lists of new and old messages for a user, are sorted by message ID. The other commands are: add_user(user_id, name),add_group(group_id,g_name), register_user(user_id,group_id), list_old_messages(user_id), list_new_messages(user_id), set_message_preview(char_count), delete_message(user_id,message_id), read_message(user_id,message_id), send_message(user_id, group_id,message), Each user command shall be followed by the abstract state space. Queries shall display results only, not the abstract state. I created this system using JAVA. I exploited the properties of "Maps", "Sets", "Lists" and "Iterators". I achieved the project goal within the given time frame and met the client requirements.

This was a great learning experience in the area of software development. I had an opportunity to learn to write acceptance tests, actual tests and use cases. I also learned that elicitating the user requirements is just as important as writing the code.
